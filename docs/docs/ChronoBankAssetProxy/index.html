ChronoBank Asset Proxy. * Proxy implements ERC20 interface and acts as a gateway to a single platform asset. Proxy adds symbol and caller(sender) when forwarding requests to platform. Every request that is made by caller first sent to the specific asset implementation contract, which then calls back to be forwarded onto platform. * Calls flow: Caller -> Proxy.func(...) -> Asset.__func(..., Caller.address) -> Proxy.__func(..., Caller.address) -> Platform.proxyFunc(..., symbol, Caller.address) * Asset implementation contract is mutable, but each user have an option to stick with old implementation, through explicit decision made in timely manner, if he doesn't agree with new rules. Each user have a possibility to upgrade to latest asset contract implementation, without the possibility to rollback. * Note: all the non constant functions return false instead of throwing in case if state change didn't happen yet.
